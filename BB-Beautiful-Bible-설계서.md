## 1. 프로젝트 개요

### 1.1 서비스 소개
- **서비스명**: BB (Beautiful Bible)
- **목적**: 아름다운 우리교회에서 보는 개역개정 성경을 온라인에서 읽을 수 있는 서비스
- **타겟 사용자**: 교인 및 성경을 읽고자 하는 모든 사용자
- **핵심 가치**: 간단하고 아름다운 성경 읽기 경험 제공

### 1.2 주요 특징
- 📱 **PWA 지원**: 앱처럼 사용 가능한 웹 서비스
- 🎨 **아름다운 디자인**: 읽기에 최적화된 깔끔한 UI
- 💰 **무료 운영**: 초기 비용 없는 정적 배포
- 📖 **개역개정**: 공식 개역개정 성경 텍스트 제공

## 2. 기술 스택 및 아키텍처

### 2.1 프론트엔드
```
Framework: Next.js 14 (App Router)
언어: TypeScript
스타일링: Tailwind CSS
상태관리: Zustand (가벼운 전역 상태)
PWA: next-pwa
아이콘: Lucide React
폰트: Pretendard (한글 최적화)
```

### 2.2 데이터
```
형식: JSON 파일 (정적)
구조: 성경책별 파일 분리
인코딩: UTF-8
크기 최적화: Gzip 압축
```

### 2.3 배포 및 호스팅
```
호스팅: Vercel (무료 티어)
CDN: Vercel Edge Network
도메인: Vercel 제공 무료 도메인 또는 커스텀 도메인
SSL: 자동 제공
```

```

## 5. 데이터 구조

### 5.1 성경 데이터 JSON 구조 (실제 데이터 기반)

#### 5.1.1 현재 데이터 형식
```json
{
  "창1:1": " 태초에 하나님이 천지를 창조하시니라",
  "창1:2": "땅이 혼돈하고 공허하며 흑암이 깊음 위에 있고 하나님의 영은 수면 위에 운행하시니라",
  "창1:3": "하나님이 이르시되 빛이 있으라 하시니 빛이 있었고",
  "창2:1": "천지와 만물이 다 이루어지니라",
  "마1:1": "아브라함과 다윗의 후손 예수 그리스도의 계보라",
  ...
}
```

#### 5.1.2 활용을 위한 데이터 파싱
```javascript
// Key 파싱 함수
function parseVerseKey(key) {
  const match = key.match(/^(\w+)(\d+):(\d+)$/);
  return {
    book: match[1],      // "창", "마" 등
    chapter: parseInt(match[2]), // 1, 2, 3 등
    verse: parseInt(match[3])    // 1, 2, 3 등
  };
}

// 책별 데이터 그룹핑
function groupByBook(bibleData) {
  const books = {};
  Object.entries(bibleData).forEach(([key, text]) => {
    const { book, chapter, verse } = parseVerseKey(key);
    if (!books[book]) books[book] = {};
    if (!books[book][chapter]) books[book][chapter] = {};
    books[book][chapter][verse] = text;
  });
  return books;
}
```

#### 5.1.3 성경책 메타데이터
```json
{
  "books": {
    "창": { "name": "창세기", "chapters": 50, "testament": "구약" },
    "출": { "name": "출애굽기", "chapters": 40, "testament": "구약" },
    "레": { "name": "레위기", "chapters": 27, "testament": "구약" },
    ...
    "마": { "name": "마태복음", "chapters": 28, "testament": "신약" },
    "막": { "name": "마가복음", "chapters": 16, "testament": "신약" },
    "눅": { "name": "누가복음", "chapters": 24, "testament": "신약" },
    ...
  }
}

### 5.2 파일 구조 (최적화된 방식)
```
/public/data/
├── bible.json                # 전체 성경 (현재 형식)
├── bible-meta.json           # 성경책 메타데이터
└── cache/                    # 런타임 캐시 (선택적)
    ├── books.json            # 책별 그룹핑 데이터
    └── chapters/             # 장별 분리 데이터
        ├── 창1.json          # 창세기 1장
        ├── 창2.json          # 창세기 2장
        └── ...
```

### 5.3 성능 최적화 전략
#### 5.3.1 초기 로딩

- **브라우저 캐싱**: localStorage에 데이터 저장
- **점진적 파싱**: 필요한 책/장만 메모리에 로드

### 5.4 데이터 최적화
- **단일 파일 로딩**: 현재 형식 활용으로 심플한 구조 유지
- **압축**: Gzip 압축으로 전송 크기 최소화  
- **캐싱**: localStorage 활용으로 재방문 시 빠른 로딩
